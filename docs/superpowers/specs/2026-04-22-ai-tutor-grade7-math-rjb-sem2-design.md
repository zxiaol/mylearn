## 1. 背景与目标

面向中小学生的学习网站，通过 AI 辅助实现“诊断 → 定向辅导 → 再测巩固”的闭环，帮助学生快速发现薄弱点并补齐。

本规格聚焦 MVP：**初一数学 + 人教版 + 七年级下册**，优先跑通学习闭环与学习进度沉淀，不做付费。

## 2. MVP 范围（明确包含 / 不包含）

### 2.1 包含

- **学生端**
  - 必须登录建档
  - 选择地区、学期（七下）、当前学习进度（到“章”）
  - 章节小测（约 20 题）
  - 薄弱点定位（粒度：**知识点 + 题型/能力点**）
  - 定向辅导闭环：**讲解 → 例题 1 道 → 练习 3 道 → 小结**
  - 掌握判定：练习 3 道 **全对** → 已掌握
  - 学习地图：按“章”展示未测/学习中/薄弱/掌握，并支持一键继续
- **家长/老师端**
  - 仅查看学习报告（只读）
  - 通过学生生成的**绑定码**建立关系
  - 报告指标（最小集）：正确率（章/知识点维度）、薄弱点列表、建议练习、用时/学习时长、连续学习天数
- **题目来源**
  - 题库优先；题库不足或需要变式时由 AI 生成，并以“可回收/可入库”的方式沉淀
- **AI 能力**
  - 接第三方大模型 API（效果优先）
  - 解答题采用“拍照上传 → 多模态模型读图判分点评”
- **内容安全（基础）**
  - 敏感/不当内容拦截（最小实现）
  - 明确学习用途提示

### 2.2 不包含（MVP 明确不做）

- 付费/订阅/次数包
- 题库后台管理系统（先用脚本/手工导入）
- 账号找回/重置密码（MVP 不提供）
- OCR 解析为文本后再判分（MVP 不做，直接多模态）
- 家长/老师端布置任务、发起专项练习/再测、催学等操作
- 错因归类（概念/计算/审题）与错题回放（可作为后续迭代）

## 3. 关键用户与使用场景

### 3.1 学生

1) 注册/登录 → 建档（地区、七下、当前章）  
2) 进入学习地图 → 选择当前章（或一键继续）  
3) 完成章节小测（约 20 题）  
4) 系统定位薄弱点（知识点+题型/能力点）  
5) 进入定向辅导闭环：讲解 → 例题 → 练习 3 道  
6) 练习 3 道全对则标记掌握，进入下一个薄弱点  
7) 薄弱点清零后，章级状态提升为“已掌握”，引导下一章或巩固

### 3.2 家长/老师

1) 登录（与学生同样的用户名+密码机制）  
2) 输入学生提供的绑定码  
3) 查看报告（周/总览），了解薄弱点与学习时长

## 4. 产品信息架构（IA）

### 4.1 学生端页面

- `/login` 登录
- `/register` 注册 + 建档（地区、学期、进度章）
- `/map` 学习地图（按章）
- `/quiz/:chapterId` 章节测验（约 20 题）
- `/review/:attemptId` 测验结果与薄弱点列表（可跳转辅导）
- `/tutor/:weaknessId` 定向辅导（讲解/例题/练习/小结）
- `/resume` 一键继续（跳转到最近未完成节点）
- `/settings` 账户信息（MVP 简化）

### 4.2 家长/老师端页面

- `/login` 登录
- `/bind` 输入绑定码
- `/report/:studentId` 学习报告（只读）

> 注：MVP 可在同一 Web 应用内通过“角色”切换显示不同导航。

## 5. 学习闭环与规则（核心业务逻辑）

### 5.1 章节范围（出题策略）

- 学生在建档/设置中选择“当前章”。
- 章节小测题目分布：
  - **主覆盖**：当前章内知识点（按题库标签与依赖图抽样）
  - **少量前置**：与当前章强相关的前置知识点（用于发现根因）

### 5.2 题型与判分

- 单选：自动判分
- 填空：自动判分（数字/分数/小数等规范化输入；MVP 可以先做较宽松的格式容错）
- 解答题：拍照上传 → 多模态模型判分
  - 输出应结构化：总分、得分点命中/缺失、关键错误位置、改进建议、参考解法

### 5.3 薄弱点识别（粒度）

薄弱点对象：\(`KnowledgePoint` + `QuestionType/Ability`\)。

MVP 规则建议（可实现为可配置策略）：
- 若在章节小测中该薄弱点命中题目数 ≥ 1 且错误率/得分低于阈值 → 标记为薄弱
- 若前置知识点相关题目错误 → 既标记当前章薄弱点，也标记前置薄弱点（用于辅导顺序排序）

### 5.4 定向辅导闭环（A）

针对每个薄弱点：
1) **讲解**：概念 + 常见误区 + 解题策略（输出应可控、短而清晰）
2) **例题 1 道**：展示标准解法/关键步骤
3) **练习 3 道**：同薄弱点、同题型/能力点的变式练习，自动/AI 判分
4) **小结**：是否掌握 + 下一步建议

掌握判定（你确认的 A）：
- 练习 3 道 **全对** → 该薄弱点标记“已掌握”
- 否则 → 继续该薄弱点新一轮辅导（题目换新/变式，避免背答案）

### 5.5 学习地图状态

章级状态（对外展示）：
- 未测：从未完成该章小测
- 学习中：小测完成但存在薄弱点未清零，或辅导未完成
- 薄弱：近期测验/辅导显示存在明显薄弱点（可与“学习中”合并为一个状态，UI 上再细分）
- 已掌握：该章关键薄弱点均已达标（MVP 可用规则：章内所有被测到的薄弱点清零 + 覆盖率达到最小门槛）

## 6. 内容与数据模型（MVP 最小可用）

> 这里按“可实现最小集”定义实体与字段范围，具体字段会在实现计划阶段进一步细化。

### 6.1 教材与知识结构

- `Book`：人教版七年级下册数学
- `Chapter`：章（学生进度选择到章）
- `KnowledgePoint`：知识点（章下细分）
- `Ability`：能力点（可选，MVP 可先与题型合并为标签）
- `QuestionType`：题型（单选/填空/解答；以及能力型题的细分标签）
- `PrerequisiteEdge`：知识点前置依赖（用于抽测前置与辅导排序）

### 6.2 题库

- `Question`
  - `id`
  - `stem`（题干）
  - `options`（可选，单选）
  - `answerKey`（标准答案/评分点）
  - `analysis`（解析）
  - `difficulty`（难度：1-5）
  - `tags`：
    - `chapterId`
    - `knowledgePointIds`
    - `questionType`
    - `abilityTags`（可选）
  - `source`：`bank` | `ai_variant`
  - `usage`：`diagnostic` | `mastery_check` | `practice`

### 6.3 用户与建档

- `User`
  - `username` + `passwordHash`
  - `role`：`student` | `guardian` | `teacher`（MVP 可先 `student/guardian`，教师同 guardian）
  - `nickname`
  - `region`
- `StudentProfile`
  - `userId`
  - `term`：七下
  - `currentChapterId`

### 6.4 学习与测验记录

- `QuizAttempt`
  - `studentId`
  - `chapterId`
  - `startedAt/finishedAt`
  - `items[]`（题目列表与答案）
  - `resultSummary`（得分、正确率、耗时）
- `Answer`
  - `questionId`
  - `type`：choice/fill/photo
  - `value`（选项/文本）
  - `photoUrls[]`（解答题）
  - `graded`（判分结果结构化）
- `Weakness`
  - `studentId`
  - `chapterId`
  - `knowledgePointId`
  - `questionType/abilityTag`
  - `status`：weak/mastered
  - `lastUpdatedAt`
- `TutoringSession`
  - `weaknessId`
  - `state`：explaining/example/practice/summary
  - `practiceAttempts[]`

### 6.5 绑定与报告

- `BindCode`
  - `studentId`
  - `code`
  - `expiresAt`（可选；MVP 可先做有效期）
- `GuardianBinding`
  - `guardianUserId`
  - `studentId`
  - `createdAt`
- `ReportSnapshot`（可选：按周聚合缓存）
  - `studentId`
  - `weekStart`
  - `metrics`（正确率、薄弱点、用时、连续天数）

## 7. AI 集成边界（可控输出约束）

### 7.1 必须结构化的 AI 输出

为保证可用性，以下输出必须为结构化 JSON（或严格可解析格式）：
- 解答题判分：分数、评分点命中、关键错误、建议、参考解法
- 生成题目：题干、答案、解析、标签（知识点/题型/难度）、可判分信息
- 辅导内容：讲解要点、例题题干/答案/解析、练习题 3 道及答案解析

### 7.2 质量与风险控制（MVP）

- 若 AI 生成题目缺少可判分答案/解析 → 丢弃或降级为“仅讲解内容”
- 对解答题判分设置失败兜底：返回“无法识别/请重拍”的可解释错误（不进行胡判）

## 8. 非功能性要求（MVP）

- **性能**：测验与辅导的交互响应在可接受范围；图片上传应有进度反馈
- **可用性**：移动端优先的响应式 Web；题目阅读与作答体验清晰
- **隐私**：存储最小必要信息；图片仅用于学习判分
- **审计（MVP 不做全量）**：仅基础拦截与提示；后续可扩展留存审计

## 9. 里程碑（面向实现计划的拆分提示）

- M1：账号体系 + 学生建档 + 章节结构导入 + 学习地图空壳
- M2：章节小测（题库组卷）+ 自动判分（单选/填空）
- M3：解答题拍照上传 + 多模态判分
- M4：薄弱点识别 + 定向辅导闭环 + 掌握判定
- M5：绑定码 + 家长/老师报告（只读）

## 10. 明确决策记录（避免歧义）

- 端形态：纯 Web（移动端自适应）
- 登录：用户名 + 密码；MVP 不做找回
- 内容：七年级下册（人教版），学生进度选择到“章”
- 章节小测：围绕当前章 + 少量前置知识
- 题型：单选/填空自动判分 + 解答题拍照多模态判分
- 辅导：讲解 → 例题 1 → 练习 3 → 小结；练习 3 全对 = 掌握
- 题目来源：题库优先 + AI 变式补足并入库
- 家长/老师端：只读报告；绑定码绑定
- 付费：MVP 不做
